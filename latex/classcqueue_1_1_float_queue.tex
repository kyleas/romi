\doxysubsection{Detailed Description}
A fast, pre-\/allocated queue of floats for Micro\+Python. 

This class is written in C for speed. When a \doxylink{classcqueue_1_1_float_queue}{Float\+Queue} object is created, memory is allocated to hold the given number of items. Data is put into the queue with its \doxylink{classcqueue_1_1_float_queue_a3d2421faddd1511caf5ba46a9971d97f}{put()} method, and the oldest available data is retrieved with the \doxylink{classcqueue_1_1_float_queue_a3beae5b8e594f3b505359fc3c04e1747}{get()} method. Because running \doxylink{classcqueue_1_1_float_queue_a3d2421faddd1511caf5ba46a9971d97f}{put()} and \doxylink{classcqueue_1_1_float_queue_a3beae5b8e594f3b505359fc3c04e1747}{get()} doesn\textquotesingle{}t allocate any memory, it can be used in interrupt callbacks.

When one creates a queue, one specifies the number of items which can be stored at once in the queue. After creating a queue, one writes items into a queue using its \doxylink{classcqueue_1_1_float_queue_a3d2421faddd1511caf5ba46a9971d97f}{put()} method. Writing into a full queue causes the oldest data to be erased; method \doxylink{classcqueue_1_1_float_queue_a31ec2169865388babb2730db7e99ee42}{full()} can be used before writing to check for such a problem. Reading from the queue is done by a call to \doxylink{classcqueue_1_1_float_queue_a3beae5b8e594f3b505359fc3c04e1747}{get()}, which returns the oldest available data item or {\ttfamily None} if the queue is empty\+:
